We start with simple logical statements. We always distinguish (as in every mathematical theorem) between the hypotheses and the statement (or assertion, which we also call goal or target). To introduce our hypotheses, we write

    If you want to prove ‚ä¢ Q and know that hPQ : P ‚Üí Q is valid, then it is sufficient to prove ‚ä¢ P (since hPQ then implies ‚ä¢ Q). In this case, apply hPQ changes the goal to ‚ä¢ P.

    false implies anything. The actual statement is ‚ä¢ false ‚Üí P. If the current target is ‚ä¢ P, and you apply the statement ‚ä¢ false ‚Üí P using apply, this is equivalent to applying exfalso.

    The two expressions False and True represent two statements that are false and true, respectively. So True should be easy to prove. This is provided by the tactic triv.

    In a proof by contradiction, instead of ‚ä¢ P, you prove the statement ‚ä¢ ¬¨P ‚Üí false (which, after intro h, leads to the assumption h : ¬¨P and the new goal ‚ä¢ false). This is logically correct, since P is true if and only if ¬¨P leads to a contradiction, i.e. an false statement. The transformation of the goal in this way is achieved with the tactic by_contra or by_contra h.

  • Exercises 01-e:

    For and and or links between statements, Lean provides the usual notations ‚àß (type \wedge) and ‚à® (type \vee). Statements linked with these connections can occur both in a hypothesis and in the goal. Now there are the following four cases:

    • ‚ä¢ P ‚àß Q: Here we must prove the two statements P and Q. With constructor exactly these two goals (with the same assumptions) are created, i.e. ‚ä¢ P and ‚ä¢ Q. If these two are shown, then obviously ‚ä¢ P ‚àß Q is also shown.

    • ‚ä¢ P ‚à® Q: To show this, it is sufficient to either show P or to show Q. In the first case, the target is replaced by ‚ä¢ P with left, and by ‚ä¢ Q with right.

    • h : P ‚àß Q: Apparently, the hypothesis h breaks down into two hypotheses, both of which must hold. Using cases' h with hP hQ, h : P ‚àß Q is transformed into two hypotheses, namely hP : P and hQ : Q.

    • h : P ‚à® Q: Similar to the last case, cases' h with hP hQ now generates two new goals, one where h : P ‚à® Q has been replaced by hP : P, and one where h : P ‚à® Q has been replaced by hQ : Q. This is logically correct, because this way you can distinguish the cases where P or Q apply.

  • Exercises 01-e:

    This is about introducing new hypotheses. With the by_cases tactic - applied to a hypothesis h : P - all possibilities are gone through that P can assume. These are that P is either true or false. So by_cases h : P introduces two new goals, one with the hypothesis h : P and one with the hypothesis h : ¬¨P.

    A very general tactic is have. Any hypotheses can be formulated here that must first be shown.
